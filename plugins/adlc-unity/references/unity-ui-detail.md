## Folder layout (default when none detected)

```
Assets/UI/
  Screens/      -- top-level UXML per screen
  Components/   -- reusable UXML fragments
  Styles/       -- all USS files
  Themes/       -- TSS + per-game theme USS
```

## UXML root pattern (full-screen panel)

```xml
<ui:UXML xmlns:ui="UnityEngine.UIElements">
  <ui:VisualElement name="safe-area-root" class="safe-area">
    <ui:VisualElement name="content" class="screen-content">
      <!-- screen body here -->
    </ui:VisualElement>
  </ui:VisualElement>
</ui:UXML>
```

Every interactive element needs `focusable="true"` and a minimum touch target of 48 dp
(`min-width: 48px; min-height: 48px` in USS). Use `flex-direction`, `align-items`,
`justify-content`, `flex-grow` for layout; never hardcode pixel positions for anything
that must scale.

## Design token USS files

`tokens-base.uss` (generated by adlc-design, do not hand-edit):

```css
/* tokens-base.uss -- generated; do not hand-edit */
:root {
  --color-primary: #E8A020;
  --color-surface: #1A1A2E;
  --spacing-md: 16px;
  --radius-card: 8px;
  --font-size-body: 14px;
}
```

Per-game brand theme (hand-authored, one per game, in `Assets/UI/Themes/`):

```css
/* theme-auto-battler.uss -- overrides tokens for this game's brand */
:root {
  --color-primary: #3A7BFF;
  --color-surface: #0D1B2A;
}
```

Theme Style Sheet wiring (`GameTheme.tss`):

```css
@import url("../../../Packages/adlc-design/tokens-base.uss");
@import url("theme-auto-battler.uss");
```

Assign `GameTheme.tss` in the Panel Settings asset (`Theme Style Sheet` field).

Applying tokens in component USS:

```css
.card {
  background-color: var(--color-surface);
  border-radius: var(--radius-card);
  padding: var(--spacing-md);
}

.cta-button {
  background-color: var(--color-primary);
  min-height: 48px;
  font-size: var(--font-size-body);
}
```

Never hardcode color hex or pixel sizes that belong to tokens. If a token is missing,
add it to adlc-design and regenerate; do not inline the value as a workaround.

## SafeAreaAdapter (MonoBehaviour)

Attach to the same GameObject as the UIDocument. Safe areas are not automatic in UI Toolkit.

```csharp
using UnityEngine;
using UnityEngine.UIElements;

[RequireComponent(typeof(UIDocument))]
public class SafeAreaAdapter : MonoBehaviour
{
    void OnEnable()
    {
        var root = GetComponent<UIDocument>().rootVisualElement;
        ApplySafeArea(root);
    }

    void ApplySafeArea(VisualElement safeAreaRoot)
    {
        var sa = Screen.safeArea;
        var panel = safeAreaRoot.panel;
        if (panel == null) return;

        // Convert screen pixels to panel points
        var minAnchor = RuntimePanelUtils.ScreenToPanel(panel,
            new Vector2(sa.xMin, Screen.height - sa.yMax));
        var maxAnchor = RuntimePanelUtils.ScreenToPanel(panel,
            new Vector2(Screen.width - sa.xMax, sa.yMin));

        safeAreaRoot.style.marginLeft   = minAnchor.x;
        safeAreaRoot.style.marginTop    = minAnchor.y;
        safeAreaRoot.style.marginRight  = maxAnchor.x;
        safeAreaRoot.style.marginBottom = maxAnchor.y;
    }
}
```

## Responsive aspect-ratio USS (no C# required)

```css
/* Tall phones (portrait 9:16 and taller) */
@media (max-aspect-ratio: 9/16) {
  .hud-bottom-bar { height: 120px; }
}

/* Wide phones / tablets (16:9 and wider) */
@media (min-aspect-ratio: 16/9) {
  .hud-bottom-bar { height: 80px; }
}
```

## Panel Settings values

| Sort Order: HUD | 0 |
| Sort Order: Popup/Modal | 10 |

| Scale Mode | Scale With Screen Size |
